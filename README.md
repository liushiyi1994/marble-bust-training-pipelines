# Marble Bust Training Pipelines

Local-first training orchestration for seven marble-bust LoRA pipelines with a hard backend split:

- `ai_toolkit` for the FLUX-family pipelines
- `diffsynth` for the Qwen, Z-Image, and FireRed pipelines

## Setup

1. Create a Python 3.12 environment.
2. Install the repo:
   `python -m pip install -e .[test]`
3. Bootstrap the pinned trainer checkouts:
   `python scripts/bootstrap_trainers.py`

## Dataset Contract

Architecture A expects:

- `manifest.json`
- `busts/*.jpg|jpeg|png`
- `busts/*.txt`
- every caption to include the trigger word

Architecture B expects:

- `manifest.json`
- `pairs/*_input.jpg`
- `pairs/*_target.jpg`
- `pairs/*.txt`
- every caption to include the trigger word

For local smoke work, keep using the demo dataset path generated by:

`python scripts/bootstrap_demo_dataset.py --output-root /tmp/demo-data --count 8 --force`

## Local Validation

Validate a pipeline by name:

`python scripts/validate.py --pipeline arch_a_klein_4b`

Validate a specific config path:

`python scripts/validate.py --config-path configs/pipelines/arch_a_klein_4b.yaml`

## Local Smoke

Run a local smoke with the backend-specific smoke entrypoint wired through the shared script:

`python scripts/smoke_test.py --pipeline arch_a_klein_4b --max-examples 10 --max-steps 100`

The current local strategy is:

- must run locally: `arch_a_klein_4b`
- try locally: `arch_a_z_image`, `arch_b_kontext_dev`
- RunPod-first: `arch_a_flux2_dev`, `arch_a_qwen_image_2512`, `arch_b_qwen_edit_2511`, `arch_b_firered_edit_1_1`

## Training

Start a training run by pipeline name:

`python scripts/train.py --pipeline arch_a_klein_4b`

Generate the backend config and prepared run layout without launching the trainer:

`python scripts/train.py --pipeline arch_a_klein_4b --dry-run`

Export the final normalized weight from a completed run:

`python scripts/export_weights.py runs/arch_a_klein_4b/<run-id>`

## RunPod

Launch through the RunPod wrapper:

`bash runpod/launch.sh arch_a_klein_4b`

The pod contract assumes:

- shared dataset volume mounted at `/workspace/shared`
- run outputs written under `/workspace/output`
- the same `scripts/train.py --pipeline <name>` entrypoint as local execution
